What a SHA is

SHA stands for "Secure Hash Algorithm." In GitHub, every time you upload or change a file, GitHub stamps that action with a long string of letters and numbers — something like a3f7b2c9e1d4.... That string is the SHA. Think of it as a receipt number or a serial number for that specific version of that specific file.

What "Copy the SHA" means

When GitHub shows you that your upload succeeded, there's usually a button or link to copy this long code. That's all "copy the SHA" means — grab that receipt number.

What it's for

In your day-to-day chart workflow, you probably don't need it at all. The SHA is useful for two situations:

  1. Rolling back a mistake. If you accidentally upload a wrong chart and overwrite the good one, the SHA lets you tell GitHub "go back to the version with this receipt number." It's your undo button.

  2. Verifying an upload worked. If you're automating uploads with a script (like through Claude Code), the script can check the SHA to confirm the file actually landed in the repo.

Bottom line for your workflow

You can safely ignore the SHA for normal operations. Your permanent URLs work regardless. The SHA is just GitHub's version tracking running in the background — it's there if you ever need to recover an older version of a chart, but you don't need to copy it or write it down during your daily routine.
